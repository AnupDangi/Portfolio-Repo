I started my journey in backend development with Node.js, doing the typical MERN stack. For a long period, I was building many personal projects and participating in hackathons. While building a major project which is still a top-notch idea and a highly capable product in the market, some questions arose. It’s a match-making and peer coding platform along with solo learning, where you get all the features of NotebookLM. (It’s paused for now as it’s a big project and too difficult to handle solo.)

But the questions that arose while building this project were: Can I make it scalable? Is WebRTC communication good? Is everything correct? When the project is big, how do you talk to core APIs and service APIs? Does any existing technology exist, such as a package or tool, which can help me rather than reimplementing things from scratch? The simple question was: am I doing something dumb?

There was one exact problem that made me look towards system design. I had implemented indexes on a table. The number of indexes was sufficient—I wanted faster retrieval. Does that mean every column should be indexed? No, right? So I might do something dumb later because I might not know the patterns of building software.

Then I looked into Gaurav Sen’s System Design playlist—it’s amazing. You will learn a lot, starting from how the fundamental architecture of different technologies works. When someone asks me how the news feed changes, how recommendations are shown to me, how the same feed changes from CDN (is it a cache?), how to retrieve it, is it single-threaded or multi-threaded—now these things click in my head. I feel like I know how things work. I know from the top level to some code level, though I don’t have experience with low-level design yet and am learning about that.

I have collected the best resources from YouTube and GitHub which I have followed and am still following to watch, take notes, and learn. I suggest if you want to know and build good, actual engineering projects, you can refer to these playlists. Even if you don’t know backend and system design, this is the best roadmap to follow.

#### The Roadmap for Anyone Who Wants to Learn Backend and System Design in 2026
Don’t think much — start with the backend playlist. Even if you think you know it, you might not know some concepts. Go with that mindset.
*   Start with Backend from First Principles
*   HLD by Gaurav Sen: both concepts and Engineering Marvel Whitepapers videos, then move to his own LLD in Java
*   If you want more, go to CodeWithAryan’s playlist, then move to questions and start practicing yourself. It all comes with practice.
